# CRAB · страница «Позиции» — README для верстальщика

Прототип: `crab-dashboard-variants.html` (рабочий файл, один экран, без сборки и зависимостей).
`crab-dashboard-standalone.html` и `index.html` — тот же файл, обёрнутый в `<!doctype html>` (standalone для раздачи, index для GitHub Pages). Живое демо: https://serjnsk.github.io/crab-dashboard-design/

Прототип — источник спецификации: токены, размеры, пороги, поведение. DOM и JS из него не переносится как есть.

---

## 1. Что в прототипе демо-специфично (НЕ верстать)

- Блок «уплотнение истории» в JS: генерирует 17 срезов к 8 ручным (итого 25), интерполяция позиций с шумом. В продакшене срезы приходят с бэка.
- Дубли групп «… · 2/3/4» — созданы для проверки sticky на длинной прокрутке.
- Детерминированные генераторы: динамика частотности (`k.wsd/k.wsed`), реклама по срезам (`k.adsAt`), позиции доп. URL (`ourUrlsFor`), выдача SERP (`serpForDate`, перестановки по seed), «старая страница» в попапе смены релеванта. Всё это — данные API.
- Хвосты вариантов дизайна: CSS-классы `.v2….v13`, легаси-хелперы (`renderSummaryTable`, `summary-strip/cards/chart`, `serpListHtml`, `nowCellHtml`, `renderDetail`, `detail-rail`) — не используются, финальный вариант один (`v15`, cells-fill).
- Переключение вариантов/тем в правом нижнем углу — служебное.

## 2. Токены и темы

- Все цвета — CSS-переменные в `:root` (строки ~4–62): `--accent`, `--bg/--surface/--surface-2`, `--border/-strong`, `--text/-2/-3`, пороги позиций `--t3/t10/t20/t50/t100` (по три: `-bg` заливка, `-fg` текст, `-heat` насыщенный), `--up/--down`, типизация `--typ-*`.
- Тёмная тема: те же переменные в `@media (prefers-color-scheme: dark)` + принудительно через `:root[data-theme="light|dark"]` (переключатель в шапке: system/light/dark).
- В тёмной теме `--t*-bg` полупрозрачные (rgba) — для липких ячеек нужна непрозрачная подложка: `background: linear-gradient(var(--t100-bg),var(--t100-bg)) var(--surface)`.
- Шрифты: системный стек (`--sans`), моно для чисел частотности (`--mono`), табличные цифры `font-variant-numeric: tabular-nums` везде в данных.

## 3. Каркас

- Топбар: высота 52px, `position:sticky; top:0; z-index:40`.
- Сайднав: 64px, `sticky; top:52px; height:calc(100vh−52px); z-index:30` (выше липких строк таблицы — флайаут мониторингов не перекрывается).
- z-index шкала: строки таблицы 2–5 → шапка таблицы 10–11 → сайднав 30 → топбар 40 → hsb (плавающий скроллбар) 60 → дропдауны 70 → поиск 80 → попапы (swap/kuPop/typMenu) 95–120 → модалки 95–96 → меню юзера 110.

## 4. Таблица позиций — адаптивность (главное)

Фиксированная левая часть: КЛЮЧ 240–340px + ТОП-10 86 + SERP 60 + ЧАСТОТНОСТЬ ~110 ≈ 490–500px.
Ширина колонки среза = (ширина контейнера − левая часть) / число срезов. Ступени:

1. **≥ 62px — полная**: позиция + дельта, дата «дд.мм» + день недели (+ время при наличии).
2. **< 62px — компакт `sl-mid`**: min-width 44px, паддинг 0 2px; позиция и дельта остаются всегда; в шапке только «дд.мм»; дельты в сводке скрываются; полные данные в title.
3. **Не влезло — `kw-clamp`**: КЛЮЧ зажимается ровно до 240px. Влезло — стоп (скролл не включаем, вертикальные sticky живы).
4. **Всё равно не влезло — `h-scroll`**: горизонтальный скролл контейнера таблицы.

**Дельты изменений не сворачиваются ни на одной ступени.**

Скролл-режим (`h-scroll`):
- Sticky left: КЛЮЧ 0 / ТОП-10 240 / SERP 326 / ЧАСТОТНОСТЬ 386. Sticky right: последний срез. Фоны липких ячеек непрозрачные.
- Старт прокрутки — у последних срезов (`scrollLeft = max`).
- Тени на краях: классы `sc-left` (scrollLeft > 2) / `sc-right` (< max−2).
- ⚠ Вертикальный sticky внутри скролл-контейнера отсчитывается от контейнера, не от окна: для thead и шапок групп в этом режиме `top:auto` (иначе шапка съезжает вниз на величину top).
- Родной скроллбар контейнера скрыт; вместо него плавающая полоса `.hsb` `position:fixed` у нижней кромки окна: left/width по контейнеру, двусторонняя синхронизация scrollLeft, видна пока таблица в вьюпорте, у конца таблицы прижимается к её нижней границе (`bottom = max(0, innerHeight − rect.bottom)`).

Пересчёт (`applySliceDensity`):
- Триггеры: рендер, resize (debounce 120мс), смена числа срезов/периода.
- ⚠ Перед измерением снять ВСЕ классы адаптации (`sl-mid`, `kw-clamp`, `h-scroll`) — иначе гистерезис: зажатый КЛЮЧ искажает ширину и решение «нужен ли скролл» становится нестабильным.
- scrollWidth читать синхронно после установки классов; ⚠ не использовать rAF — не срабатывает в фоновых вкладках.
- Высота thead переменная (время в датах, компакт) — мерить фактическую → `--thead-h`; липкие шапки групп: `top: calc(52px + var(--thead-h))`.

Сводка живёт в той же колоночной сетке (первый tbody) — ступени применяются автоматически.

## 5. Ячейка позиции

- `.cell` — кнопка на всю ячейку (cells-fill: заливка цветом порога `--t*-bg` + текст `--t*-fg`), клик открывает SERP-модалку.
- Дельта — «степенью» сверху справа: 9.5px, 700, `--up/--down`; знак у отрицательных из числа.
- Маркеры — колонкой у правого края ПОСЛЕ позиции: ⇅ сверху, реклама (SVG-логотип Google Ads) снизу; круглые бейджи 16px (14px в компакте) с подложкой `--surface` и рамкой; пара центрируется по вертикали, одиночный — по центру; hover ⇅ — scale(1.12).
- ⇅ кликабелен → попап «Смена релевантной страницы»: Было/Стало, URL+тайтл, у каждого копирование и переход; позиционирование у иконки, переворот вверх при нехватке места.
- «—» вне ТОП-100, «н/д» позиция не снята (пунктирная рамка).
- title ячейки: дата, позиция, дельта.

## 6. Колонка КЛЮЧ

- Иконка ПС — ссылка `https://yandex.ru/search/?text=<encoded kw>`, `target=_blank`.
- Клик по тексту ключа или иконке копирования (появляется на hover строки, 18px) — копирование в буфер; фидбек: галочка `--up` 1.2с; фолбэк `execCommand('copy')`.
- Релевантная страница: строка «≠ /path/» под ключом; если сайт в выдаче несколькими URL — рядом чип «N URL ▾» → попап `kuPop`: все URL с актуальными позициями последнего среза, позиции подсвечены порогами.

## 7. Частотность

- Показывается ОДИН вид: переключатель «базовая | !точная» в шапке колонки (сегмент как «123/%» в сводке). Для Google — один вид, переключателя нет.
- Формат: ≥1000 → `1,3k`, ≥1e6 → `1,2M`; точная с оранжевым префиксом `!`.
- Дельта к прошлому месяцу — «степенью», формат тот же (`+12,3k`); полное число в title. Месяцы — `WS_MONTH`/`WS_PREV`.

## 8. SERP-модалка

- Шапка: ПС + ключ + крестик; период (общий календарный компонент: диапазон + частота) + пресеты «Последние 5/10» (active — когда набор совпадает). Чипов отдельных дат нет: диапазон+частота определяют показанные срезы.
- Колонки по 310px; ОДИН срез — колонка 600px по центру (ширина результатов Google), на узких экранах `min(600px, 92vw)`.
- **Одна общая вертикальная прокрутка** на все колонки (`.sm-cols{overflow:auto}`), шапки колонок sticky top:0.
- **Фиксированные высоты** — позиции разных дат не разъезжаются: шапка колонки 38px, полоса рекламы 30px (есть во ВСЕХ колонках; без рекламы — заглушка «Без рекламы в выдаче»), строка результата 74px (тайтл клампится на 2 строки).
- Строка: маленький номер позиции 17px с цветом типизации → фавиконка 19px (акцент) в строке тайтла → тайтл → строка URL. Фавиконка двухслойная: цветная буква-заглушка (hsl от hash домена) мгновенно + реальная `google.com/s2/favicons?domain=…&sz=32` поверх после onload.
- Наш сайт: только красная вертикальная черта слева 3px, без заливки и меток.
- ТОП-10 по умолчанию; если наша позиция 11–20 — ТОП-20.
- Кросс-подсветка: hover по строке подсвечивает этот домен во всех колонках (класс `hl`, фон `--t3-bg`).
- ПКМ по строке — меню «Категория сайта»: пункты из настроек типизации с цветными точками, галочка у текущей, выбор перекрашивает бейджи домена во всех колонках; внизу «Настроить категории…».
- Числа позиций «наша позиция» в шапке колонки — бейдж с цветом порога.

## 9. Прочие компоненты

- Календарь периода: открывается ПОД своей кнопкой, правым краем к ней; прижим к краям окна, переворот вверх при нехватке места; общий для тулбара и модалки (в модалке — fixed, чтобы не резался overflow).
- «Позиции конкурента»: модалка 400px с затемнением — поле адреса (автофокус), «Показать позиции» (primary) / «Отмена»; пустое поле — красная рамка; Enter/Escape/бекдроп.
- Меню юзера (hover по аватару): шапка имя+почта, строка «Команда», две плитки-счётчика (видит проектов / создал), секции «Аккаунт»/«Данные» с иконками, красный «Выйти». Тарифов и лимитов нет.
- Флаги регионов — SVG (эмодзи-флаги не рендерятся на Windows): в статусах заголовка и после названий супергрупп, 17×12, тултип с регионом.
- «Перейти к группе» — дропдаун с поиском, скроллом к группе и подсветкой-вспышкой.
- Все оверлеи закрываются: клик вне, Escape; одновременно открыт максимум один.

## 10. Брейкпоинты

- ≥1200px: таблица без скролл-контейнера (sticky живут), кроме принудительного `h-scroll`.
- <1200px: контейнер `overflow-x:auto`, вертикальные sticky таблицы отключены (`top:auto`).
- <900px: мобильный паттерн (спарклайн вместо колонок срезов) — в прототипе НЕ реализован, только описан.

## 11. Что подключается к API (сейчас захардкожено/генерится)

срезы и позиции; частотности обоих видов + их дельты и месяц данных; события смены релеванта (старый/новый URL+тайтл); признак рекламы и число объявлений по срезу; выдача ТОП-10/20 с тайтлами/URL/типизацией; все URL сайта по ключу с позициями; категории типизации проекта (названия/цвета); команда и счётчики проектов юзера; регионы супергрупп; фавиконки (сервис или свой прокси).
